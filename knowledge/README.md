# Knowledge Management System (PARA Method)

Based on Tiago Forte's PARA system for organizing digital information.

## Folder Structure

```
knowledge/
├── 01_Projects/          # Active work with deadlines/goals
│   ├── whitelabl_seo/
│   ├── job_search_automation/
│   └── filmmaker_career/
├── 02_Areas/             # Ongoing responsibilities (no deadline)
│   ├── health/
│   ├── finances/
│   └── relationships/
├── 03_Resources/         # Reference material
│   ├── tools/
│   ├── processes/
│   └── inspiration/
├── 04_Archives/          # Completed/inactive items
└── daily_notes/          # Daily capture
```

## PARA Definitions

**Projects**: Active work with a deadline or specific outcome
- Examples: Whitelabl SEO campaign, Job search automation setup
- Has: deadline, specific goal, actionable tasks

**Areas**: Ongoing responsibilities without deadline
- Examples: Health maintenance, Financial planning, Relationships
- Has: standard to maintain, no end date

**Resources**: Reference material for future use
- Examples: Tool tutorials, Process docs, Inspiration
- Has: useful information, no immediate action needed

**Archives**: Completed projects or inactive areas
- Examples: Finished campaigns, Old jobs
- Has: historical value, no current relevance

## Daily Note Template

See: `daily_notes/YYYY-MM-DD.md`

## Nightly Review Process

1. Read today's conversation
2. Extract key information (decisions, actions, learnings)
3. File into appropriate PARA folder
4. Update project/task lists
5. Archive completed items
